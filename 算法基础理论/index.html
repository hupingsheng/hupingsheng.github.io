高精度

大整数在数组中如何存储的?

  • int表示是无法存下来的，所以拆分每一个存到数组中

  • 数字的末位存储在数组中的低位（a[0],a[1]…）

  • 数组相加会有进位，因此在数组的末尾添加一个进位更容易操作。

在数组中的阅读顺序相反，输出的时候要将数组中的数字逆序输出

tips:实际上和平时处理一个整数的每一位顺序是一样的，先求余，后整除，都是先处理末位

前缀和

一维前缀和

  • 定义：数组中前i个数的和，即si=a1+a2+a3+...+ais_i = a_1 + a_2 + a_3 + ... + a_isi​=a1​+a2​+a3​+...+ai​

  • 如何求：

    前缀和 S[i] = a[1] + a[2] + … a[i]

    前缀和实际上是从下标1开始的

  • 作用：

    快速求出原数组a中一段区间[l,r]的和---->一次运算求出任意区间内的和O(1)O(1)O(1)

差分

一维差分

  • 定义

    数组 a1 a2 a3… an

    构造 b1 b2 b3… bn

    st ai = b1 + b2 + … + bi 即a数组是b数组的前缀和, b是a的差分 差分与前缀和互逆

  • 如何构造

    **结论：给a数组中的[l, r]区间中的每一个数都加上c,只需对差分数组b做 b[l] + = c, b[r+1] -= c。**时间复杂度为O(1)O(1)O(1), 大大提高了效率。

  • 作用：

    如果在原数组a上的某些给定区间进行一系列操作，求原数组a，则给定差分数组b，利用差分数组可以在O(1)O(1)O(1)内完成，在O(n)O(n)O(n)下求出原数组a.

  • 个人感觉有点==蝴蝶传播效应==那味了，比如在元素组的【l, r】区间内都加上c，那么只需要在差分数组的l索引的位置上+c即可，因为差分数组的前缀和就是原数组，所以差分数组累加，则 l 索引后面的每一个位置都会+c，直接传播下去；然后在差分数组的 r+1 位置上 -c，则原数组从r + 1的位置往后都会 -c，所以两者相互抵消了，最后体现在原数组上就是正好【l, r】区间 +c 。
