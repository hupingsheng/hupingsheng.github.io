20. 有效的括号

题目主要信息:

  • 给定一个只包含大中小左右括号的字符串，判断其中括号是否合法
  • 大中小括号的数学顺序与合法无关，只需要每种左括号在右边有相应匹配的右括号即可，不可交叉匹配，应该是括号嵌套

全程遍历完，不出错就是有效

分析出出错的可能情况即可

  1. 左边括号多余。已经遍历完了字符串，但是栈不为空，说明有相应的左括号没有右括号来匹配，所以return false
  2. 右边括号多余。遍历字符串匹配的过程中，栈已经为空了，没有匹配的字符了，说明右括号没有找到对应的左括号return false
  3. 不多余，但是左右不匹配，比如[ ) return false

剑指 Offer 40. 最小的k个数

题目主要信息：

  • 对于一个给定无序数组，返回最小的k个元素，顺序任意
  • k和数组有特殊情况需要单独讨论

priority_queue ，又称优先队列 ，是一个容器----允许在$$O(log_N)$$时间复杂度下插入数据，在$$O(1)$$时间复杂度下取得容器的最大（最小）值

  • 时间复杂度 $$O(nlog_k)$$ 构建和维护大小$$K$$为的堆，需要$$log_2K$$，加上遍历整个数组

  • 空间复杂度：$$O(k)$$，堆空间为$$K$$个元素
