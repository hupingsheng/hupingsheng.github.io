差分

一维差分

  • 定义

    数组 a1 a2 a3… an

    构造 b1 b2 b3… bn

    st ai = b1 + b2 + … + bi 即a数组是b数组的前缀和, b是a的差分 差分与前缀和互逆

  • 如何构造

    **结论：给a数组中的[l, r]区间中的每一个数都加上c,只需对差分数组b做 b[l] + = c, b[r+1] -= c。**时间复杂度为O(1)O(1)O(1), 大大提高了效率。

  • 作用：

    如果在原数组a上的某些给定区间进行一系列操作，求原数组a，则给定差分数组b，利用差分数组可以在O(1)O(1)O(1)内完成，在O(n)O(n)O(n)下求出原数组a.

  • 个人感觉有点==蝴蝶传播效应==那味了，比如在元素组的【l, r】区间内都加上c，那么只需要在差分数组的l索引的位置上+c即可，因为差分数组的前缀和就是原数组，所以差分数组累加，则 l 索引后面的每一个位置都会+c，直接传播下去；然后在差分数组的 r+1 位置上 -c，则原数组从r + 1的位置往后都会 -c，所以两者相互抵消了，最后体现在原数组上就是正好【l, r】区间 +c 。

求最大值

本质：队列中的元素必须比新入队尾的元素大，因为同一队列快照中，队尾元素一定比队头元素存活的久【属于同一时代】，那么既然队尾元素存活的久，元素又大，那么选取最大值，以队尾元素为代表就好了，没必要用到队头元素。故队头元素若小，则需要弹出来。

同理，如果求滑动窗口内的最小值的话，如果当前的值比队尾的值还要小的话，那么队列的部分值将永远无法用到，那么不如直接弹出吧！

最长公共子序列

a[i]，b[j]选或不选，是否出现在公共子序列中进行分类，共四种情况

  • a[i]不选，b[i]不选===>a[i]，b[j]一定出现在子序列中 ==>f[i][j] = f[i - 1][j - 1]

  • a[i]不选，b[i]选====>a[i]不出现在子序列中，b[j]出现在子序列中

  • a[i]选，b[i]不选====> a[i]出现在子序列在，b[j]不出现在子序列中

  • a[i]选，b[i]选====>a[i]和b[j]都不出现在子序列中 f[i][j] = f[i - 1][j - 1] + 1

第2和3的情况，不好表示，直接写f[i][j] = f[i - 1][j]此时的f[i - 1][j]未必一定是以b[j]结尾的；f[i][j] = f[i][j - 1]，此时的f[i - 1][j]未必一定是以a[i]结尾的。

但是可以得出的是，f[i - 1][j]是严格包含第一种和第二种情况的，f[i][j - 1]是严格包含第一种和第三种情况的。

所以一二三种情况直接用f[i][j] = max(f[i - 1][j], f[i][j - 1])来概括，

而第四种则是用f[i][j] = f[i - 1][j - 1] + 1来明确表示。

最短编辑距离

  • A[i]删除后，A和B能保持一致，则A[i - 1] 与B[j]是相等的，即f[i][j] = f[i - 1][j] + 1
  • 若A[i]的后面添加A[i + 1]后，A和B能保持一致，则A[i]与B[j - 1]是相等的，即f[i][j] = f[i][j-1] + 1
  • 若A[i - 1]和B[j - 1]是相等的，则A[i]和B[i]直接匹配，不需要操作直接一致，，即f[i][j] = f[i - 1][j - 1]
  • 若A[i - 1]和B[j - 1]是不相等的，则A[i]需要修改一下，A[i]和B[j]才能匹配，即f[i][j] = f[i - 1][j - 1] + 1

时间复杂度：O(3n2)O(3n^2)O(3n2)
分析：DP问题的时间复杂度，为状态数量 * 转移的计算量 ，状态数量为n2n^2n2，每次状态转移的计算量为比较增删改的最小值，三次比较计算，所以，时间成本为3n23n^23n2
