第一章 Linux起源

第二章 系统分区

  1. 磁盘分区是使用分区编辑器（partition editor）在磁盘上划分几个逻辑部分。碟片一旦划分成数个分区（Partition），不同类的目录与文件可以存储进不同的分区。

  2. 格式化（高级格式化）又称逻辑格式化，它是指根据用户选定的文件系统（如FAT16、FAT32、NTFS、EXT2、EXT3、EXT4等），在磁盘的特定区域写入特定数据，在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间。将硬盘分成一个个等大小的数据块，同时建立inode。当查找文件时，是按照inode来找到文件的条目，从而找到这个文件保存在哪几个数据块中，进而找出这几个数据块，拼凑成一个完整文件

  3. 硬件设备文件名

    Linux中所有的硬件设备都是文件

  4. 挂载

    挂载点（类似windows中的盘符）

​ Linux中用空的文件名称作为盘符

  • 必须分区

    • swap分区（交换分区，内存两倍，不超过2GB）

    • / (根分区）

  • 推荐分区

    • /boot (启动分区，200MB) 保存启动时需要的数据
  1. 总结

​ 分区：把大硬盘分为小的逻辑分区
​ 格式化：写入文件系统，同时把硬盘中数据全部删除
​ 分区设备文件名：给每个分区定义设备文件名
​ 挂载：给每个分区分配挂载点

学习Linux注意事项

  • Linux严格区分大小写

  • Linux中所有内容以文件形式保存（命令行的更改只是临时生效，所有的更改必须通过文件形式更改才有效）

  • Linux不靠扩展名区分文件类型(即没有扩展名概念)，实际上是靠文件权限区分的

    • 压缩包：.gz、.bz2 .tar.bz2、.tgz等
    • 二进制软件包：.rpm
    • 网页文件：.html、.php
    • 脚本文件：.sh
    • *配置文件：.conf

    扩展名只是为了使管理员更加方便区分的，实际上不需要取这些后缀的，系统可以识别的

  • Linux所有的存储设备都必须挂载（设备文件名和盘符连接起来的过程称挂载）之后用户才能使用，包括硬盘、U盘和光盘

  • Windows下的程序不能直接在Linux中安装和运行

服务器注意事项

  • 远程服务器不允许关机，只能重启
  • 重启时应该关闭服务
  • 不要在服务器访问高峰运行高负载命令
  • 远程配置防火墙（过滤器，符合规则要求则允许访问）时不要把自己踢出服务器

3.1 文件处理命令

3.1.1 命令格式与目录处理命令ls

  1. 命令格式：命令[-选项][参数]

    • 选项：做不同的操作，或者命令结果呈现不同的信息形式【简单说：调整功能的】
    • 参数： 命令对谁进行操作的（用户、进程、文件、目录）
    • 说明：
      • 个别命令使用不遵循此格式
      • 当有多个选项时，可以写在一起 【-l -a等于-la 合并顺序无所谓】
      • 简化选项与完整选项 【 -a等于--all】

  2. 目录处理命令

    • 命令名称：ls
    • 命令英文原意：list
    • 命令所在路径：/bin/ls
    • 执行权限：所有用户
    • 功能描述：显示目录文件
    • 语法：ls 选项[-ald][文件或目录]
      • -a 显示所有文件 all，包含隐藏文件.xxx
      • -l 详细信息显示 long
      • -d 查看当前目录本身，不显示目录下的数据
      • -h 人性化显示处理，如文件大小 由默认字节转化为KB显示
      • -i 查看i节点

    查询结果解读

    文件类型（- 二进制文件 d 目录 l 软链接文件）

    权限： r 读 w写 x执行

    最小权限原则：权限够用就好，即使是文件所有者，也不一定需要执行权限x 。比如日志文件，读写足够，只有一些脚本程序，需要运行才给执行权限

3.2.1 权限管理命令chmod

  • 权限管理命令 ： chmod

  • 命令名称：chmod

  • 命令英文原意：change the permissions mode of a file

  • 命令所在路径：/bin/chmod

  • 执行权限：所有用户

  • 语法：chmod [{ugoa}{+-=}{rwx}] [文件或目录]
    [mode=421 ] [文件或目录] r ---- 4 w ---- 2 x ---- 1 ---------常用方式

    • ​ -R 递归修改 修改目录及其目录下所有文件

  • 功能描述：改变文件或目录权限

    易错点：删除一个文件的条件，是对这个文件的所在目录有写权限

    所以在Linux系统中，一般目录的r x权限一般是同时出现的

第五章 Linux软件安装

5.1 软件包管理简介

  1. 软件包分类

    • 源码包（脚本安装包，例如c源码包）

    • 二进制包（RPM包、系统默认包）

  2. 源码包

    源码包的优点：

    • 开源，如果有足够的能力，可以修改源代码
    • 可以自由选择所需的功能
    • 软件是编译安装，所以更加适合自己的系统，更加稳定也效率更高
    • 卸载方便

    源码包的缺点：

    • 安装过程步骤较多，尤其安装较大的软件集合时（如LAMP环境搭建），容易出现拼写错误
    • 编译过程时间较长，安装比二进制安装时间长
    • 因为是编译安装，安装过程中一旦报错新手很难解决

  3. RPM包

    二进制包的优点：

    • 包管理系统简单，只通过几个命令就可以实现包的安装、升级、查询和卸载

    • 安装速度比源码包安装快的多

    二进制包缺点：

    • 经过编译，不再可以看到源代码

    • 功能选择不如源码包灵活

    • 依赖性（安装过程由于依赖性，手动安装很繁琐）

5.2.1 包命名与依赖性

  1. RPM包命名原则

  2. RPM包依赖性

    • 树形依赖： a→b→c（安装时则需要按c、b、a的顺序安装，卸载时按a、b、c顺序卸载）
    • 环形依赖： a→b→c→a（安装包abc用一条命令同时安装，就会解决）
    • 模块依赖： 模块依赖查询网站：www.rpmfind.net

5.4源码包管理

5.4.1 源码包和RPM包的区别

  1. 区别

    • 安装之前的区别：概念上的区别
    • 安装之后的区别：安装位置不同

  2. RPM包安装位置

    • 是安装在默认位置中

  3. 源码包安装位置

    • 安装在指定位置当中，一般是/usr/local/软件名/ 这是系统给准备的第三方软件位置

  4. 安装位置不同带来的影响

    • RPM包安装的服务可以使用系统服务管理命令（service）来管理，例如RPM包安装的apache的启动方法是：
      • /etc/rc.d/init.d/httpd start（启动方法一）
      • service mysqld start（启动方法二）
    • 源码包安装的服务则不能被服务管理命令管理，因为没有安装到默认路径中。所以只能用绝对路径进行服务的管理，如：
      • /usr/local/apache2/bin/apachectl start

5.4.2 源码包安装过程

  1. 安装准备

    • 安装C语言编译器gcc yum -y install gcc

    • 下载源码包

  2. 安装注意事项

    • 源代码保存位置：/usr/local/src/
    • 软件安装位置：/usr/local/ 类似windows中C盘的program
    • 如何确定安装过程报错：
      • 安装过程停止
      • 并出现error、warning或no的提示

  3. 源码包安装过程

    • 下载源码包

    • 解压缩下载的源码包

    • 进入解压缩目录

      • 查看INSTLL文件

  4. 源码包卸载

    不需要卸载命令，直接删除安装目录即可。不会遗留任何垃圾文件

5.5 脚本安装包

  • 脚本安装包并不是独立的软件包类型，常见安装的是源码包
  • 是人为把安装过程写成了自动安装的脚本，只要执行脚本，定义简单的参数，就可以完成安装。
  • 非常类似于Windows下软件的安装方式
